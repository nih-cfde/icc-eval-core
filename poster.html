Gathering Insights Into CFDE Projects

Integration and Coordination Center (ICC)

Motivation

The ICC is tasked with (among other things) gathering and presenting information about CFDE projects. We want to allow project groups and the NIH to easily see the activity and impact of their work in a simple and centralized way.

The info we gather currently includes publications, software stats, and website metrics. These come from a variety of sources, take many forms and sizes, and can be sensitive or private, which presents several challenges that led us to develop a unique and flexible software architecture to handle them.

Challenges

Frequency
Need to gather info ideally daily
Availability
APIs available for some info, but some need webpage scraping
Scale
Large # of projects, queries can take a long time
Complexity
Info is complex, dense, and different structures and formats
Simplicity
Project groups don't have time to set up complex integration
Transparency
Everyone should be able to see how all info is gathered
Access
Some info should be restricted to certain people/groups

Technologies

TypeScript
to maintain full end-to-end type safety, from gathering to displaying
Node
to run gathering process and other parts of pipeline
Vue
to create live web dashboard (better than re-emailing PDFs)
GitHub Actions
to automatically run pipeline, and test and deploy dashboard
Various tools
to parse CSV, PDF, etc.
Playwright
to scrape HTML pages (and print dashboard as PDFs as needed)
Custom tools
to auto-parallelize and cache (w/ TTL) all network requests
API Clients
to ease gathering from e.g. GitHub and Google Analytics
